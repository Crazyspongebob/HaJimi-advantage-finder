# Gallup Theme: Restorative (排难)

## Core Definition
Adept at dealing with problems. You are good at figuring out what is wrong and resolving it.

---

## [Mode A] Chatty Mode (闲聊模式)

### UG Scenario (本科场景)
- **Topic 1:** 修理的“强迫症”。电脑蓝屏了或者是寝室空调坏了，你是那个只会打电话叫维修工的，还是那个会忍不住先拆开看个究竟、研究出是哪根线松了、并尝试亲手把它修好的人？ `[Signal: Diagnostic drive]`
- **Topic 2:** 漏洞的“眼中钉”。小组作业出现了一个极难解决的逻辑漏洞，大家都打算绕过去不提了，你会不会非要钻牛角尖、甚至推倒重来也要把这个漏洞补上？ `[Signal: Defect intolerance]`
- **Topic 3:** 历史遗留难题。如果你在社团里发现一个一直以来的‘老大难’问题（比如经费总是对不齐），你会即便这不归你管、也会主动请缨去把它‘理顺’吗？ `[Signal: Legacy problem reclamation]`
- **Topic 4:** “救火队长”的特质。你会因为这种‘总想把坏的变好’的性格而感到心累吗？比如你看到身边的朋友生活一团糟，你会忍不住去帮他收拾烂摊子？ `[Signal: Fixer syndrome]`
- **Topic 5:** 妙手回春的快感。你会迷恋那种‘起死回生’的故事吗？或者你会特别享受那种把一个支离破碎的项目救回来带来的爽感吗？ `[Signal: Redemption joy]`
- **Topic 6:** 修复即创造。你更倾向于认为：一个成功的人生，不仅在于你创造了多少，更在于你修复了多少受损的事物？ `[Signal: Maintenance-first logic]`

### PG Scenario (研究生场景)
- **Topic 1:** 智力“维修家”。实验室里有一台已经报废了三年的旧仪器，大家都要扔了，你会不会非要在那研究它的电路板、查阅远古资料，直到有一天它奇迹般地重新转起来？ `[Signal: Obsolescence renewal]`
- **Topic 2:** 与Bug的纠缠。当你的代码运行到第9999行时突然报错，你是感到沮丧，还是会有一种‘终于来活了’的莫名兴奋，非要抓出那个Bug才肯罢休？ `[Signal: Debugging obsession]`
- **Topic 3:** 挑战悬而未决的“沉疴”。你会发现自己特别擅长分析‘为什么之前失败了’吗？对于那种悬而未决的沉疴难题（如十几年的学术僵局），你是否有一种去挑战它的冲动？ `[Signal: Root cause analysis]`
- **Topic 4:** 沉没成本的挽救者。你会为了补救一个数据不理想的课题、而投入比开新题多得多的精力吗？ `[Signal: Sunk cost reclaimer]`
- **Topic 5:** 抢救及格线。你会不会觉得，那种‘把事情从0分改到80分’带来的快感，比‘从90分改到100分’更高？ `[Signal: Basic functionality restoration]`
- **Topic 6:** 问题终结者。你梦想过有一天能成为一个被全行业尊称为‘问题终结者’的存在吗？ `[Signal: Expert troubleshooter]`

## [Mode B] Scale Mode (量表模式 - 1-5)
*1=极其不同意, 2=不太同意, 3=一般, 4=比较同意, 5=极其同意*

1. 我天生就对解决各种复杂的技术性或逻辑性难题充满了兴趣。 [ ]
2. 我非常享受发现、分析并修复错误的过程。 [ ]
3. 我认为修补受损的系统或关系，比重新开始要有意义得多。 [ ]
4. 看到原本无法运转的事物在我的努力下恢复正常，我会感到很有成就感。 [ ]
5. 很多人认为我是一个非常出色的“救火队员”或“问题专家”。 [ ]
6. 与其做那些一帆风顺的事情，我更喜欢挑战那些充满了故障和漏洞的复杂局面。 [ ]
7. 我对各种事物的内在机制和运作逻辑有强烈的好奇心。 [ ]
8. 对我来说，一个无解的Bug是让我保持长久思考的最好动力。 [ ]
9. 我能迅速地辨别出一个方案中那个最隐秘且致命的缺陷。 [ ]
10. “药到病除”和“亡羊补牢”是我应对挑战的核心策略。 [ ]
11. (反向) 我认为修补旧东西太浪费时间了，有问题不如直接买新的。 [ ]
12. (反向) 碰到难以解决的困难，我更容易选择绕过它，追求那些容易成功的事。 [ ]
13. (反向) 我对于分析失败的原因没有任何兴趣。 [ ]
14. (反向) 相比起修补破损，我更喜欢在一张白纸上进行全新的创作。 [ ]
15. (反向) 我觉得那些整天折腾故障的人其实是活得太辛苦了。 [ ]

## [Module C] Signal Interpretation (信号解读)
- **正面特征**：极高冷静度、卓越的诊断能力、能显著降低组织故障带来的损失。适合：运维工程师、医生、调试专家、危机公关补救、资深审计、历史修复专家。
- **负面风险**：可能会过于聚焦问题而显得悲观、可能会在没有故障的稳健期感到无聊而人为制造变数、可能会让队友感到被过分挑剔。
- **岗位匹配暗示**：适合需要“精准修复”和“后端保障”的赛道。
