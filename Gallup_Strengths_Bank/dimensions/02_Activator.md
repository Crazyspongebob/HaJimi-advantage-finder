# Gallup Theme: Activator (行动)

## Core Definition
"When can we start?" is a recurring question. Only action is real to you.

---

## [Mode A] Chatty Mode (闲聊模式)

### UG Scenario (本科场景)
- **Topic 1:** 团队开会还在争论方案细节（比如租教室还是拉赞助）时，你是否常有种“别说了，我先出去试一把”的冲动？聊法：比起反复列计划，你是不是更愿意先走出去试错，哪怕方案还不完美？ `[Signal: Bias for action]`
- **Topic 2:** 面对新比赛或新机会时的“入场”节奏。当别人都在观望、想等细则更清楚点时，你是否已经把名报了，打算“先上车后补票”？聊聊你最近一次这种“不等了，先开始”的经历。 `[Signal: Early entry]`
- **Topic 3:** 突然想去某家店或做某件事的决策速度。你是那种穿上鞋就走的果断派，还是必须要搜遍全网攻略、研究一小时评价才动弹的稳健派？分享一个你“说走就走”或“纠结很久”的小故事。 `[Signal: Spontaneity]`
- **Topic 4:** 面对不熟悉的软件或智能设备。你是习惯先读完说明书，还是直接乱点一气直到发现它的逻辑？你是否觉得“动手摸索”比“坐着学习”更有效？ `[Signal: Learning through doing]`
- **Topic 5:** 讨论陷入僵局、大家都不说话时的反应。你会为了打破沉默而提一个哪怕不怎么成熟的想法吗？你是否觉得“即便错误的行动也总好过原地踏步”？ `[Signal: Breaking deadlock]`
- **Topic 6:** 从Idea诞生到告诉别人的“思维-行动延迟”。你会为了拿出一份完美PPT而闭关一个月，还是倾向于在五分钟内直接发条微信开启合作？谈谈你的想法转化速度。 `[Signal: Thought-to-Action latency]`

### PG Scenario (研究生场景)
- **Topic 1:** 新课题刚有个初步想法时的第一反应。你是那种“先调两组数据跑跑看”的实践派，还是非要把技术路线图画到完美、找导师确认三遍才敢动手的谨慎派？ `[Signal: Low entry threshold]`
- **Topic 2:** 实验卡壳时的应对策略。你更倾向于“先换个参数试试看结果”的试错法，还是坐在工位上对着白板推演两小时逻辑？聊聊你更相信行动还是推演。 `[Signal: Experimental speed]`
- **Topic 3:** 导师刚提了一个新思路，你会多久开始尝试？会是在散会后的半小时内就开始搜代码库、部署工具包吗？谈谈你对这种“快速响应”反馈的沉迷程度。 `[Signal: Execution reflex]`
- **Topic 4:** 面对繁琐的行政审批或报销流程。你是不是那个最等不了审批、宁愿自己垫钱先把零件买了把实验跑通的人？那种“为了效率优先行动”的选择对你来说频繁吗？ `[Signal: Resource agility]`
- **Topic 5:** 当合作方迟迟不给数据或反馈时的直接行动。你会选择继续发写封得体的正式邮件，还是倾向于直接打电话甚至冲到对方办公室问个究竟？ `[Signal: Impatience with lag]`
- **Topic 6:** 关于“清醒感”的来源。你是否觉得坐在工位上思考反而让你焦虑，只有真正动起手来（写代码、做实验）时，你的脑子才是最清醒的？聊聊你的“行动觉醒”。 `[Signal: Action-based clarity]`

## [Mode B] Scale Mode (量表模式 - 1-5)
*1=极其不同意, 2=不太同意, 3=一般, 4=比较同意, 5=极其同意*

1. 我认为与其过度分析，不如先采取行动来观察结果。 [ ]
2. 我经常感到一种必须“现在就开始”的紧迫感。 [ ]
3. 在团队中，我通常是第一个说“让我们开始吧”的人。 [ ]
4. 我认为通过亲身实践学到的东西比仅仅听讲要深刻得多。 [ ]
5. 过多的讨论和会议常常让我感到不耐烦，我只想干活。 [ ]
6. 我不害怕犯错，因为我觉得错误本身就是有价值的反馈。 [ ]
7. 我能迅速地激发周围人的能量，让他们也动起来。 [ ]
8. 我通常在没有完全把握的情况下，也会勇敢地迈出第一步。 [ ]
9. 对我来说，只有已经执行的代码/方案才是真实存在的。 [ ]
10. 我天生就有一种把“想法”瞬间转化为“现实”的冲动。 [ ]
11. (反向) 我认为在没有周密计划之前，贸然行动是极其不负责任的。 [ ]
12. (反向) 我更喜欢观察别人先做，自己最后再动手，以避免风险。 [ ]
13. (反向) 我觉得那些急躁的人往往会把事情搞砸。 [ ]
14. (反向) 慢工出细活是我的座右铭，我可以等很久再开始。 [ ]
15. (反向) 我通常需要别人从后边推我一把，我才会开始新的尝试。 [ ]

## [Module C] Signal Interpretation (信号解读)
- **正面特征**：极高行动力、初创团队的核心驱动器。适合：市场外勤、早期项目负责人、公关应急。
- **负面风险**：容易鲁莽、可能忽略细节风险、会让慢节奏的队友感到压力过大。
- **岗位匹配暗示**：适合需要“快速响应”和“破冰”的环节，避免需要“超长周期调研”的岗位。
